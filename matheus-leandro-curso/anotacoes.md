# Spring
## CACARTERÍSTICAS
* Inversão de Controle (IoC): O próprio Spring gerencia a criação e ciclo de vida dos objetos, evitando que você tenha que instanciálos manualmente
* Injeção de Dependências (DI): Você pode definir dependências entre as classes sem precisar instanciar objetos diretamente. Torna o código mais modular e testável
* Programação Orientada a Aspectos: Você pode adicionar funcionalidades como logs, segurança e transações de banco de dados sem precisar modificar diretamente seu código

# SpringBoot
## Características
* Autoconfiguração
* Servidor embutido: Você pode rodar suas aplicações como um simples arquivo .jar, sem precisar de um servidor próprio
* Starter Packs: pacotes de dependência já prontos pra facilitar a integração com BD's, mensageria, etc

## Onde é usado?
* É a principal escolha para desenvolvimento de API's, sistemas empresariais e microsserviços em Java.
* Aplicações Monolíticas
* Microservices. Se integra perfeitamente com o Spring Cloud, que já oferece algumas ferramentas
* API's REST
* Cloud Computing

## PRÉ-REQUISITO PARA AS AULAS QUE FALTAM
* Conceitos de API REST
